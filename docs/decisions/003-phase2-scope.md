# ADR-003: Phase 2 のスコープ定義

## ステータス
承認

## コンテキスト
Phase 1（Webhook + キーワード応答 + コメントDM + コンタクト管理）が完了。
次にManyChatの中核機能であるシナリオ配信と、ユーザーがGUIで操作できる管理画面を構築する。

## 決定
Phase 2 は以下の2大機能に絞る:

### 1. シナリオ配信（ステップDM）
- シナリオ: 複数のステップ（メッセージ）を時系列で順番に送信する仕組み
- トリガー: キーワードマッチ or コメントDM or API呼び出しでシナリオ開始
- ステップ: テキストメッセージ + 遅延（例: 1時間後、1日後、3日後）
- 条件分岐: タグの有無で送信をスキップ（シンプル版）
- 進行管理: contact × scenario の進行状態をD1で管理
- Cron実行: 5分ごとに配信予定のステップを確認・送信

### 2. 管理画面（Next.js 15 + Cloudflare Pages）
- ダッシュボード: コンタクト数、直近の配信数、アクティブシナリオ数
- キーワードルール管理: 一覧・作成・編集・削除
- コメントトリガー管理: 一覧・作成・編集・削除
- シナリオビルダー: シナリオの作成・ステップ追加・遅延設定
- コンタクト一覧: タグフィルター・検索
- ADMIN_API_KEY認証でWorker APIと通信

## 理由
- シナリオ配信はManyChatの最も価値の高い機能。これがないとManyChat代替を名乗れない
- 管理画面がないとCLI/API操作のみになり、非エンジニアが使えない
- Phase 1のCRUD APIがあるので、管理画面はそれをGUIで叩くだけ

### Phase 2 に含めないもの
- AI自動応答 → Phase 3
- MCP対応 → Phase 3
- マルチプラットフォーム → Phase 4
- A/Bテスト → Future

## 影響
- Phase 2完了で「非エンジニアでもブラウザからシナリオを組んで自動DMを配信できる」状態になる
- これがMVP完成ライン
