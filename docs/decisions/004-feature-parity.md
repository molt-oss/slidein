# ADR-004: 競合分析に基づく不足機能の追加

## ステータス
承認

## コンテキスト
LINE Harness（42テーブル、100+エンドポイント）とManyChat（業界最大手）を分析し、slideinに不足している機能を特定。Phase 3としてまとめて実装する。

## 決定
Phase 3 で以下の高優先度機能を実装する:

### バッチ1: コア機能拡張
1. **ブロードキャスト** — 全コンタクト/タグ/セグメント宛の一括DM送信（即時 or 予約）
2. **スコアリング** — 行動ベースのリードスコア（メッセージ開封、リンククリック、キーワード応答）
3. **IF-THEN 自動化ルール** — イベントトリガー × アクション（タグ追加、シナリオ開始、DM送信等）
4. **テンプレート変数** — {{name}}, {{username}} 等のパーソナライズ

### バッチ2: マーケティング機能
5. **トラッキングリンク** — URL短縮 + クリック計測 + 自動タグ付け
6. **配信時間帯制御** — 指定時間帯のみ配信（深夜送信防止）
7. **Webhook OUT** — 外部サービスへのイベント通知（Stripe, Slack等）
8. **CV計測** — コンバージョンポイント定義 → イベント記録

### バッチ3: AI & 拡張
9. **AI自動応答** — Claude API連携（ナレッジベースからの自然言語応答）
10. **MCP対応** — Claude Code から全機能を自然言語で操作

## 理由
- ブロードキャスト、スコアリング、自動化はLINE HarnessとManyChatの両方にある必須機能
- これらが揃えばManyChat代替としてREADMEの機能比較表で完全に勝てる
- AI + MCPはLINE Harnessにもない差別化ポイント
