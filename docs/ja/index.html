事前準備

始める前に、以下のものを用意してね 👇

📸

Instagram ビジネスまたはクリエイターアカウント

個人アカウントだとAPIが使えないよ。Instagramの設定 → アカウント → プロアカウントに切り替え → ビジネスかクリエイターを選んでね。

👤

Facebook アカウント

Meta Developer Portalで必要だよ。 facebook.com

☁️

Cloudflare アカウント（無料）

無料プランのWorkers + D1だけでOK！ サインアップ →

🟢

Node.js 20+ & pnpm

nodejs.org · pnpmのインストール: npm install -g pnpm

1

クローン & インストール

bash 
git clone https://github.com/molt-oss/slidein.git
cd slidein
pnpm install
2

Cloudflare セットアップ

2.1 — Wrangler にログイン

bash 
npx wrangler login

2.2 — D1 データベースを作成

bash 
npx wrangler d1 create slidein-db

📋 出力された database_id をコピーして、apps/worker/wrangler.toml に貼り付けてね:

database_id = "YOUR_D1_DATABASE_ID"  ← ここを置き換えるよ

2.3 — マイグレーション実行

全6つのマイグレーションを順番に実行して、データベースのスキーマをセットアップするよ:

bash 
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0001_initial_schema.sql
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0002_pending_messages.sql
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0003_scenarios.sql
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0004_phase3_batch1.sql
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0005_phase3_batch2.sql
npx wrangler d1 execute slidein-db --remote --file=packages/db/migrations/0006_phase3_batch3.sql
3

Meta アプリ設定

3.1 — 開発者として登録

developers.facebook.com にアクセスして、まだの人はFacebookアカウントを開発者として登録してね。

3.2 — 新しいアプリを作成

  1. Create App をクリック
  2. ユースケース → "Other" を選択
  3. アプリタイプ → "Business" を選択
  4. アプリ名（例: slidein-myapp）と連絡先メールアドレスを入力
  5. Create App をクリック

3.3 — Instagram APIを追加

アプリのダッシュボードで「Add Products」の中にある Instagram を見つけて、Set Up をクリックしてね。

3.4 — Instagram テスターを追加

  1. 左のサイドバーから App Roles → Roles に移動
  2. Instagram Testers の下にある Add Instagram Testers をクリック
  3. 自分のInstagramユーザー名を入力
  4. スマホでInstagramを開く → 設定 → ウェブサイト許可 → テスター招待 → 承認

3.5 — 認証情報をゲットしよう

認証情報 見つかる場所
App Secret App Settings → Basic → App Secret（Showをクリック）
Access Token Instagram → API Setup → Generate Token（自分のIGアカウントを選択）
Instagram Account ID Instagram → API Setup → 「Instagram Account ID」として表示されてるよ
Verify Token 好きなランダム文字列でOK（例: my-slidein-verify-2024）
4

Worker デプロイ

4.1 — シークレットを設定

各コマンドを実行して、聞かれたら対応する値を貼り付けてね:

bash 
npx wrangler secret put META_APP_SECRET
npx wrangler secret put META_ACCESS_TOKEN
npx wrangler secret put META_VERIFY_TOKEN
npx wrangler secret put ADMIN_API_KEY
npx wrangler secret put IG_ACCOUNT_ID

💡 ADMIN_API_KEY はダッシュボードがWorkerと通信するためのキーだよ。ランダムな文字列を生成してね: openssl rand -hex 32

4.2 — デプロイ

bash 
npx wrangler deploy --config apps/worker/wrangler.toml

📋 出力されるURLをメモしておいてね — こんな感じ:
https://slidein-worker.YOUR_SUBDOMAIN.workers.dev
次のステップで使うよ！

5

Webhook 接続

  1. Meta アプリの左サイドバーから Webhooks に移動
  2. ドロップダウンから Instagram を選択
  3. Subscribe to this object をクリック（既に設定済みならEdit）

  4. 以下を入力してね:

    Callback URL: https://slidein-worker.YOUR_SUBDOMAIN.workers.dev/webhook
    Verify Token: META_VERIFY_TOKEN に設定した値
  5. Verify and Save をクリック
  6. 以下のフィールドをSubscribeしてね:
    messages messaging_postbacks comments
6

ダッシュボード

管理ダッシュボードをローカルでセットアップして起動するよ:

bash 
cd apps/web
cp .env.local.example .env.local

.env.local を自分の値で編集してね:

.env.local 
API_URL=https://slidein-worker.YOUR_SUBDOMAIN.workers.dev
API_KEY=your-admin-api-key-here

そしたら開発サーバーを起動！

bash 
pnpm dev
7

テストしよう！ 🎉

1

ダッシュボードを開いて、キーワードルールを作成するよ:

キーワード: hello → レスポンス: Hey! Thanks for reaching out 👋

2

Instagramを開いて、自分のビジネスアカウントに "hello" とDMを送ってみてね（別のアカウントから、またはAPIテストツールを使ってね）。

3

自動返信が来たら — セットアップ完了！ 🚀

トラブルシューティング

問題 解決方法
Webhook の検証が失敗する Workerのシークレットに設定した META_VERIFY_TOKEN と、Metaコンソールで入力した値が完全に一致してるか確認してね。
DMの自動返信が来ない Webhookフィールドで messages をSubscribeしてるか確認！あとInstagramアカウントがビジネス/クリエイターアカウントになってるかもチェックしてね。
D1 マイグレーションエラー wrangler.tomlの YOUR_D1_DATABASE_ID を実際のデータベースIDに置き換えたか確認してね。
ダッシュボードで「Unauthorized」が出る .env.localの API_KEY がWorkerの ADMIN_API_KEY シークレットと一致してるか確認してね。
トークンが期限切れ/無効 Instagramのアクセストークンは期限切れになることがあるよ。Meta Developer Console → Instagram → API Setup から再生成してね。
wrangler が見つからない npx wrangler を使ってね（dev dependenciesに含まれてるよ）。またはグローバルインストール: npm i -g wrangler
